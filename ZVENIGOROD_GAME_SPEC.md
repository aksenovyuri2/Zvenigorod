# Звенигород: Симулятор Мэра
## Техническое задание для Claude Code

---

## 1. Концепция

**Жанр:** Пошаговая экономико-политическая симуляция (single-player)

**Сеттинг:** Реальный город Звенигород, Московская область. Игрок — новый мэр, избранный на 5 лет (20 кварталов). Стартовые условия приближены к реальным: население ~25 000, исторический город с монастырём, близость к Москве (50 км), слабая инфраструктура, хорошая экология.

**Цель:** Сделать Звенигород самым привлекательным городом для жизни на планете. Привлекательность измеряется через удовлетворённость пяти групп населения. Финальный результат — позиция в глобальном рейтинге городов (стартовая позиция ~8500 из 10000).

**Ключевой игровой цикл:**
1. Игрок видит текущее состояние города (метрики, бюджет, население, рейтинг)
2. Происходит случайное событие (не каждый ход)
3. Игрок выбирает 2 решения из 5-6 предложенных (ограничен бюджетом)
4. Движок обрабатывает решения → обновляет метрики → пересчитывает удовлетворённость групп → обновляет население и бюджет → обновляет глобальный рейтинг
5. Игрок видит результаты хода (дельты всех показателей)
6. Переход к следующему кварталу

**Настроение:** Серьёзная, но не мрачная симуляция. Без мемов. С юмором в событиях, но реалистичными экономическими последствиями. Текст — на русском языке.

## 3. Экономическая модель

### 3.1. Метрики города (8 штук)

Каждая метрика — число от 0 до 100. Стартовые значения отражают реальный Звенигород:

| Метрика | Старт | Описание |
|---|---|---|
| infrastructure | 35 | Дороги, водопровод, ЖКХ, транспорт |
| ecology | 65 | Чистота воздуха, зелень, переработка отходов |
| culture | 45 | Достопримечательности, мероприятия, досуг |
| digital | 20 | Интернет, цифровые сервисы, умный город |
| safety | 40 | Безопасность, пожарные, МЧС, освещение |
| healthcare | 35 | Медицина, поликлиники, скорая, профилактика |
| education | 40 | Школы, детсады, библиотеки, курсы |
| economy | 30 | Бизнес-среда, рабочие места, налоговая база |

**Затухание (decay):** Каждый ход все метрики естественно снижаются на 1-2 пункта (износ, амортизация). Это создаёт давление: нельзя один раз построить и забыть, надо поддерживать. Формула decay:
```
decay = max(1, floor(metric / 40))
```
Т.е. чем выше метрика, тем дороже её поддерживать.

### 3.2. Группы населения (5 штук)

Каждая группа имеет свою формулу удовлетворённости, зависящую от метрик:

**Пожилые (elderly):**
- healthcare × 0.30 + safety × 0.25 + ecology × 0.20 + culture × 0.15 + infrastructure × 0.10

**Фрилансеры (freelancers):**
- digital × 0.30 + culture × 0.20 + ecology × 0.20 + economy × 0.15 + infrastructure × 0.15

**Семьи с детьми (families):**
- education × 0.30 + safety × 0.25 + healthcare × 0.20 + infrastructure × 0.15 + ecology × 0.10

**Молодёжь (youth):**
- culture × 0.25 + digital × 0.25 + economy × 0.20 + education × 0.20 + ecology × 0.10

**Бизнес (business):**
- economy × 0.30 + infrastructure × 0.25 + digital × 0.20 + safety × 0.15 + education × 0.10

### 3.3. Бюджет

**Стартовый бюджет:** 850 млн руб.

**Доход за квартал (автоматический):**
```
revenue = 400 (федеральные трансферты)
        + population × 0.012 (НДФЛ и местные налоги)
        + economy × 2.0 (налоги от бизнеса)
        + culture × 1.2 (туризм)
```

**Расходы:** Определяются выбранными решениями. Каждое решение имеет фиксированную стоимость.

**Обязательные расходы (каждый ход):**
```
mandatory = 200 + population × 0.005
```
Это зарплаты чиновников, содержание инфраструктуры, коммуналка.

**Долг:** Если бюджет уходит в минус, образуется долг. Долг начисляет 5% процентов каждый ход. Если долг > 500 — появляется штраф ко всем метрикам (-2 каждый ход). Если долг > 1000 — игра заканчивается досрочно (дефолт).

### 3.4. Население

**Стартовое:** 25 000

**Прирост/убыль за ход:**
```
avg_satisfaction = среднее удовлетворённости всех 5 групп
if avg_satisfaction > 60:
    migration = (avg_satisfaction - 60) × 15 + random(-50, 50)
elif avg_satisfaction < 40:
    migration = (avg_satisfaction - 40) × 20 + random(-50, 50)
else:
    migration = random(-100, 100)

natural_growth = population × 0.001

population += migration + natural_growth
```

Население влияет на доходы, но и на обязательные расходы. Быстрый рост без инфраструктуры может создать проблемы.

### 3.5. Глобальный рейтинг

Рейтинг от 1 до 10000 (1 = лучший город мира). Формула:

```
avg_metrics = среднее всех 8 метрик
pop_factor = min(1.0, population / 100000)
score = avg_metrics × 0.7 + pop_factor × 15 + avg_satisfaction × 0.3
rank = max(1, round(10000 - score × 100))
```

**Целевые ориентиры для игрока:**
- Старт: ~8500
- Хорошо: <5000 (лучшая половина)
- Отлично: <1000 (топ-10%)
- Победа: <100 (элита)
- Идеал: #1

## 4. Система решений

### 4.1. Механика выбора

Каждый ход игроку предлагается **5-6 решений**, из которых можно выбрать **до 2-х** (ограничение — бюджет и лимит решений). Решения выбираются из пула ~30 решений с учётом:
- Не показывать `once`-решения, которые уже приняты
- Ротация: каждый ход — новый набор, не повторять 2 хода подряд
- Валидация: решение доступно только если бюджет позволяет

### 4.2. Типы решений

**Разовые (once: true):** Можно принять один раз за игру. После принятия исчезают из пула. Обычно крупные инфраструктурные проекты.

**Повторяемые:** Можно принимать многократно. Эффект накапливается, но с diminishing returns: при повторном применении эффект = 70% от предыдущего.

**С рекуррентным эффектом (recurring):** Решение имеет постоянный эффект на бюджет в последующие ходы (например, налоговые льготы снижают доход каждый ход).

### 4.3. Каталог решений (минимум 30 штук)

Каждое решение должно содержать:
```
{
  id: string,              // уникальный идентификатор
  name: string,            // короткое название (до 40 символов)
  desc: string,            // описание на 1-2 предложения
  cost: number,            // стоимость в млн руб
  effects: {               // изменения метрик
    [metric]: number       // может быть отрицательным
  },
  once?: boolean,          // можно принять только 1 раз
  recurring?: number,      // постоянное изменение бюджета
  requires?: {             // условия доступности
    [metric]: number       // минимальное значение метрики
  },
  unlocks?: string[],      // id решений, которые становятся доступны после этого
  populationEffect?: number // прямой эффект на население
}
```

**Примеры решений по категориям:**

**Инфраструктура:**

**Экология:**

## 5. Система событий

### 5.1. Механика

Каждый ход с вероятностью **40%** происходит случайное событие. Событие показывается игроку перед фазой принятия решений. Некоторые события дают выбор (event_choice), большинство — автоматические.

### 5.2. Типы событий

**Автоматические (без выбора):**
```
{
  id: string,
  text: string,           // описание (1-2 предложения с эмодзи в начале)
  effects: { [metric]: number },
  budget: number,          // изменение бюджета
  population?: number      // изменение населения
}
```

**С выбором:**
```
{
  id: string,
  text: string,
  choices: [
    {
      label: string,       // текст кнопки выбора
      effects: { [metric]: number },
      budget: number,
      population?: number
    }
  ]
}
```

### 5.3. Каталог событий (минимум 20 штук)

**Природные:**
- Паводок затопил улицы (infra-5, safety-3, budget-40)
- Аномальные морозы повредили водопровод (infra-6, healthcare-2, budget-60)
- Жаркое лето — засуха (ecology-4, healthcare-2, budget-20)
- Ураган повалил деревья (ecology-3, infra-2, safety-2, budget-30)

**Экономические:**
- Федеральный грант на развитие малых городов (budget+150)
- Инвестор хочет построить завод [ВЫБОР: принять/отказать]
- Подрядчики подняли цены (budget-50)
- Крупная компания хочет открыть офис — требует digital>50 [ВЫБОР]
- Рост цен на стройматериалы — все строительные решения дорожают на 20% на 3 хода

**Социальные:**
- Блогер снял видео о Звенигороде (culture+4, economy+5, population+600)
- Местная школа выиграла олимпиаду (education+4, culture+2)
- Протест жителей против стройки [ВЫБОР: уступить/продолжить]
- Миграция из Москвы — удалёнщики ищут жильё (population+400, requires digital>40)
- Эпидемия гриппа (healthcare-4, budget-25)

**Политические:**
- Губернатор предлагает объединить Звенигород с соседним посёлком [ВЫБОР]
- Проверка прокуратуры — штраф за нарушения (budget-80, safety-2)
- Выборы в городскую думу — нужен рейтинг одобрения >50% или штраф к метрикам

**Культурные:**
- Съёмочная группа снимает фильм (culture+5, economy+3, budget+20)
- Экологи нашли редких птиц — статус экозоны (ecology+6)
- Известный архитектор предлагает бесплатно спроектировать парк (culture+5, ecology+3)

### 5.4. Цепочки событий

Некоторые события должны быть связаны. Пример:
1. Ход 5: «Инвестор предлагает завод» → если принял
2. Ход 8: «Жители жалуются на выбросы» → ecology-5
3. Ход 12: «Завод загрязнил реку» → ecology-10, healthcare-5, budget-100

Реализация: поле `triggers` в событии, которое добавляет будущие события в очередь.

## 6. Фазы игры

### 6.1. Экран стартового меню
- Название игры: «Звенигород: Симулятор Мэра»
- Краткое описание: «Вы — новый мэр Звенигорода. У вас 5 лет, чтобы превратить тихий подмосковный город в лучшее место для жизни на планете.»

- Текст события крупным шрифтом

### 6.3. Фаза принятия решений
**Левая панель (состояние города):**
- Квартал X из 20 (прогресс-бар)
- Бюджет (текущий, доход за ход, обязательные расходы)
- Долг (если есть, красным)
- Население
- Глобальный рейтинг (с трендом ↑↓)

- 5-6 карточек решений

- Карточки кликабельны, выбранные подсвечиваются
- Если бюджета не хватает — карточка серая, неактивная
- Счётчик: «Выбрано X из 2»
- Кнопка «Завершить ход» (активна когда выбрано 1-2 решения, или «Пропустить ход» если 0)

### 6.4. Фаза результатов хода
- Анимация/переход
- Показать дельты всех метрик (зелёные +, красные -)
- Показать изменение населения
- Показать бюджет: доход — расходы = остаток
- Показать изменение глобального рейтинга
- Показать изменение удовлетворённости каждой группы
- Кнопка «Следующий квартал»

### 6.5. Экран конца игры
- Срок закончился (20 ходов) или дефолт
- Финальный рейтинг + оценка (буква: S/A/B/C/D/F)
- Финальные метрики vs стартовые
- График динамики рейтинга за всю игру (recharts LineChart)
- График динамики населения
- Текстовый итог: достижения и провалы
- Кнопка «Играть снова»

## 8. Состояние игры (state)

```typescript
interface GameState {
  turn: number;                    // 0-20
  phase: 'start' | 'event' | 'decisions' | 'results' | 'end';
  budget: number;                  // текущий бюджет (млн руб)
  debt: number;                    // долг (если бюджет < 0)
  population: number;
  metrics: {
    infrastructure: number;
    ecology: number;
    culture: number;
    digital: number;
    safety: number;
    healthcare: number;
    education: number;
    economy: number;
  };
  prevMetrics: Metrics;            // метрики на начало хода (для отображения дельт)
  selectedDecisions: string[];     // id выбранных решений текущего хода
  availableDecisions: Decision[];  // решения, предложенные на текущий ход
  usedOnceDecisions: string[];     // id решений с once=true, уже принятых
  decisionHistory: {               // id решений, принятых за все ходы (для diminishing returns)
    [id: string]: number;          // количество раз принято
  };
  recurringEffects: {              // активные рекуррентные эффекты
    budget: number;
  };
  currentEvent: Event | null;      // событие текущего хода
  eventQueue: QueuedEvent[];       // запланированные цепочечные события
  history: TurnSnapshot[];         // снапшоты каждого хода для графиков
  globalRank: number;
  turnRevenue: number;
  turnExpenses: number;
}

interface TurnSnapshot {
  turn: number;
  metrics: Metrics;
  population: number;
  budget: number;
  globalRank: number;
  satisfaction: { [group: string]: number };
}
```

## 9. Алгоритм обработки хода

```
function processTurn(state, selectedDecisionIds):
  1. Сохранить prevMetrics = copy(state.metrics)
  
  2. Применить decay ко всем метрикам:
     for each metric:
       metric -= max(1, floor(metric / 40))
  
  3. Применить эффекты выбранных решений:
     for each decision:
       multiplier = 1.0
       if decision принималось ранее:
         multiplier = 0.7 ^ (количество_раз_ранее)
       for each effect:
         metric += effect × multiplier
       budget -= decision.cost
       if decision.recurring:
         recurringEffects.budget += decision.recurring
       if decision.once:
         usedOnceDecisions.add(decision.id)
       decisionHistory[decision.id] += 1
  
  4. Применить рекуррентные эффекты:
     budget += recurringEffects.budget
  
  5. Вычесть обязательные расходы:
     budget -= 200 + population × 0.005
  
  6. Прибавить доход:
     revenue = calcRevenue(population, metrics)
     budget += revenue
  
  7. Обработать долг:
     if budget < 0:
       debt += abs(budget)
       budget = 0
     debt *= 1.05 (5% процент)
     if debt > 0 and budget > 0:
       payment = min(budget, debt)
       budget -= payment
       debt -= payment
     if debt > 500:
       all metrics -= 2
     if debt > 1000:
       phase = 'end' (дефолт)
  
  8. Clamp все метрики в [0, 100]
  
  9. Рассчитать удовлетворённость групп
  
  10. Рассчитать миграцию населения
  
  11. Рассчитать глобальный рейтинг
  
  12. Проверить eventQueue на запланированные события для следующего хода
  
  13. Сгенерировать случайное событие с вероятностью 40% (если нет запланированного)
  
  14. Сохранить snapshot в history
  
  15. turn += 1
  
  16. if turn >= 20: phase = 'end'
      else if есть событие: phase = 'event'
      else: phase = 'decisions'
```

## 10. Генерация доступных решений на ход

```
function generateAvailableDecisions(state):
  1. Отфильтровать из ALL_DECISIONS:
     - Убрать once-решения из usedOnceDecisions
     - Убрать решения, у которых не выполнены requires
     - Убрать решения дороже budget + 50 (можно показывать, но неактивными)
  
  2. Из оставшихся выбрать 6 решений:
     - Минимум 1 дешёвое (cost <= 40)
     - Минимум 1 с trade-off (имеет отрицательные эффекты)
     - Минимум 1 из категории с самой низкой метрикой
     - Остальные — случайные, но с ротацией (не повторять предыдущий ход)
  
  3. Отсортировать по стоимости (от дешёвых к дорогим)
```

## 11. Рандом

Использовать seeded PRNG для воспроизводимости (опционально — показать seed игроку):

```javascript
function seededRandom(seed) {
  let s = seed;
  return () => {
    s = (s * 16807) % 2147483647;
    return (s - 1) / 2147483646;
  };
}
```

При старте игры генерировать seed = Date.now(). Seed используется для:
- Выбора событий
- Выбора доступных решений
- Вариации миграции населения

## 12. Балансировка

### 12.1. Тестовые сценарии для валидации баланса

**Сценарий A (оптимальная игра):** Игрок всегда выбирает решения с наибольшим суммарным эффектом. Ожидаемый результат: рейтинг 200-500 к 20-му ходу.

**Сценарий B (случайные решения):** Игрок выбирает случайно. Ожидаемый результат: рейтинг 4000-6000.

**Сценарий C (бездействие):** Игрок пропускает все ходы. Ожидаемый результат: рейтинг ухудшается до 9000+ из-за decay.

**Сценарий D (только экономика):** Все решения — economy. Ожидаемый результат: бюджет растёт, но рейтинг средний (3000-4000) из-за перекоса.

### 12.2. Ключевые правила баланса
- Невозможно максимизировать все метрики: decay + бюджет не позволяют
- Trade-off решения дают больше суммарных очков, но с риском
- Диверсифицированная стратегия должна побеждать узкую
- Долг — ловушка: легко попасть, трудно выбраться
- К 20 ходу при хорошей игре бюджет должен быть положительным, но не огромным
