## ИЗМЕНЕНИЕ: Срок игры и структура

Игра теперь длится **два срока по 5 лет = 40 ходов максимум**. После первого срока (20 ходов) — выборы. Если проиграл — game over. Если выиграл — второй срок (ещё 20 ходов), после которого финальный экран.

## 1. ЖИВОЙ РЕЙТИНГ С РЕАЛЬНЫМИ ГОРОДАМИ

### 1.1 Концепция

Вместо абстрактного числа рейтинг показывает таблицу реальных городов мира. Игрок видит 3 города выше и 3 города ниже Звенигорода. При прокрутке — полный список.

### 1.2 База городов (100 штук, ранжированы по привлекательности)

Города имеют фиксированный score (0–100). Score Звенигорода вычисляется динамически, и город вставляется в рейтинг по позиции.

```typescript
interface CityEntry {
  name: string;       // "Вена, Австрия"
  flag: string;       // "🇦🇹"
  score: number;      // 0–100, фиксированный
  icon: string;       // характерная эмодзи (🏛, 🌊, 🗽...)
  tagline: string;    // "Столица музыки и кофе"
}
```

### 1.4 Score Звенигорода

```
zvenigorod_score = average(все 8 метрик) × 0.85 + population_bonus + diversity_bonus

population_bonus = min(10, population / 10000)
diversity_bonus = min(5, (min_satisfaction / max_satisfaction) × 5)
// diversity_bonus поощряет равномерное развитие для всех групп
```

**Стартовый score: ~38** (помещает Звенигород между Лагосом и Карачи).

## 3. ВИЗУАЛИЗАЦИЯ ГОРОДА

### 3.1 Концепция

Изометрическая pixel-art карта Звенигорода, состоящая из тайлов. Карта отражает текущее состояние города: при улучшении метрик появляются/улучшаются здания, при деградации — ухудшаются.

### 3.2 Зоны карты (сетка 8×6 = 48 тайлов)

```
┌──────────────────────────────────────────────┐
│  [лес]  [лес]  [лес]  [река] [река] [река]   │  ← природная зона
│  [лес]  [парк] [жилой] [набер] [мост] [поле] │  
│  [поле] [школа] [жилой] [центр] [церковь] [лес] │  ← историческое ядро
│  [ферма] [жилой] [дорога] [админ] [больн] [жилой] │  ← городской центр
│  [поле] [завод] [дорога] [рынок] [жилой] [парк] │  ← промзона / торговля
│  [лес]  [жд]   [дорога] [жилой] [коворк] [лес]  │  ← транспортный узел
└──────────────────────────────────────────────┘
```

### 3.3 Визуальные состояния тайлов

Каждый тайл имеет 3–5 визуальных уровней, зависящих от связанных метрик:

```typescript
interface TileVisual {
  id: string;
  name: string;
  linkedMetrics: MetricKey[];    // какие метрики влияют
  levels: TileLevel[];           // от худшего к лучшему
}

interface TileLevel {
  minScore: number;              // минимальный средний score связанных метрик
  sprite: string;                // CSS-класс или SVG-id
  description: string;           // тултип
}
```

Примеры:

**Тайл «Дорога»** (infrastructure):
- `< 25`: разбитая грунтовка, ямы
- `25–50`: потрёпанный асфальт
- `50–75`: нормальная дорога, разметка
- `> 75`: идеальная дорога, велодорожка, освещение

**Тайл «Парк»** (ecology + culture):
- `< 30`: заброшенный пустырь, мусор
- `30–50`: простой сквер, пара лавочек
- `50–70`: ухоженный парк, деревья, фонтан
- `> 70`: европейский парк, арт-объекты, кафе

**Тайл «Жилой квартал»** (infrastructure + economy):
- `< 30`: хрущёвки с облупленными фасадами

## 4. ОНБОРДИНГ И ТУЛТИПЫ

### 4.1 Стартовый онбординг (4 шага)

При первом запуске — пошаговый туториал поверх интерфейса (overlay с подсветкой элемента):

**Шаг 1: «Добро пожаловать, мэр!»**
> Вы — новый мэр Звенигорода, маленького подмосковного города с богатой историей, но скромными показателями. Ваша задача — за два срока (10 лет) превратить его в лучший город планеты.

**Шаг 2: Подсветка метрик**
> Это 8 показателей вашего города. Они определяют, насколько разным группам жителей нравится здесь жить. Без внимания метрики деградируют — город не стоит на месте.

**Шаг 3: Подсветка решений**
> Каждый квартал вы выбираете 2–3 решения. Они стоят денег и влияют на метрики. Баланс бюджета — ваша ответственность. Банкротство = конец игры.

**Шаг 4: Подсветка рейтинга**
> Вот ваше место среди городов мира. Обгоняйте реальные города — от Саратова до Вены. Удачи, мэр!

### 4.2 Контекстные подсказки (первые 3 хода)

- **Ход 1:** При наведении на карточку решения — пульсирующий тултип «Нажмите, чтобы выбрать. Вы можете выбрать до 3 решений.»
- **Ход 2:** При появлении события — тултип «События происходят случайно. Некоторые дают выбор — думайте внимательно.»
- **Ход 3:** При первом отрицательном изменении метрики — тултип «Метрики деградируют без внимания! Не забывайте о слабых показателях.»

### 4.3 Тултипы на всех элементах

Тултипы появляются при hover (desktop) или long-press (mobile). Задержка: 300ms.

| Элемент | Тултип |
|---|---|
| Каждая метрика (бар) | Название + описание + что на неё влияет + «Важно для: [группы]» |
| Каждая группа населения | Название + от чего зависит удовлетворённость (веса) |
| Бюджет | «Квартальный бюджет. Доход зависит от населения и экономики.» |
| Население | «Люди переезжают в города с высокой привлекательностью и уезжают из плохих.» |
| Рейтинг | «Ваше место среди 100 реальных городов мира. Цель — #1!» |
| Стоимость решения | «Будет списано из бюджета в конце хода.» |
| Эффект решения (+N) | «Метрика увеличится на N пунктов.» |
| Трейдофф (-N) | «Внимание: это решение снизит показатель!» (красный) |
| Значок «once» | «Можно выбрать только один раз за игру.» |
| Значок «unlock» | «Доступно, когда [условие].» |
| Долг | «Растёт на 5% каждый квартал. Долг > 500 — штрафы. > 1000 — банкротство.» |

### 4.4 Реализация

```typescript
interface Tooltip {

```

Компонент `<Tooltip>` — портал, рендерится поверх всего. Позиционирование: `getBoundingClientRect()` + offset.

## 5. УГЛУБЛЕНИЕ ГЕЙМПЛЕЯ

### 5.1 Рейтинг одобрения мэра (approval)

Новый ключевой показатель: **одобрение мэра** (0–100). Стартовое значение: 60.

```
approval_change = 
  + (attractiveness_delta × 0.5)       // рост привлекательности = одобрение
  + event_approval_effects              // события влияют на одобрение
  - (debt > 200 ? 3 : 0)               // долг снижает одобрение
  - (any_metric_below_20 ? 2 : 0)      // критически низкая метрика = недовольство
  + (milestone_reached ? 5 : 0)        // достижение milestone = всплеск одобрения
  + random(-2, 2)                       // шум
```

Approval влияет на:
- Доступность некоторых решений (approval < 30 — «Дума блокирует ваши решения», доступно только 1 решение вместо 2–3)
- Результат выборов (раздел 6)
- Некоторые события (при approval < 25 — «Митинг против мэра», approval -5)

## 6. МЕХАНИКА ВЫБОРОВ

### 6.1 Когда происходят выборы

Выборы — после хода 20 (конец первого 5-летнего срока). Если мэр побеждает — играет ещё 20 ходов (второй срок). Второй срок = финальный.

### 6.2 Формула результата выборов

```
win_probability = 
    approval × 0.40                         // текущее одобрение — главный фактор
  + attractiveness_growth × 0.25            // рост привлекательности за срок
  + population_growth_percent × 0.15        // рост населения в %
  + no_bankruptcy_bonus × 0.10              // не было долга > 500
  + random_factor × 0.10                    // удача

// нормализовать к 0–100
// если win_probability >= 50 — победа
```

Где:
- `attractiveness_growth` = (текущий attractiveness - стартовый) / стартовый × 100
- `population_growth_percent` = (текущее население - 25000) / 25000 × 100
- `no_bankruptcy_bonus` = 100 если долг никогда не превышал 500, иначе 0
- `random_factor` = random(0, 100)

### 6.3 Экран выборов

Многоэтапный экран:

**Этап 1: Предвыборная кампания**

```

│                                                    │

│  достойны ли вы второго срока.                     │
│                                                    │
│  📊 Ваши показатели:                               │
│  Одобрение: 67%                                    │
│  Рост привлекательности: +45%                      │
│  Население: 25,000 → 34,200 (+37%)                │
│  Бюджет: без критичных долгов ✅                    │
│                                                    │
│  Ваш оппонент: Дмитрий Козлов                      │

```

**Этап 2: Подсчёт голосов (анимация)**

Анимированный прогресс-бар: «За вас: 54.3% | За оппонента: 45.7%». Счётчик крутится 3–5 секунд, потом фиксируется.
